What Load is

Load is an offline-first iPad workspace for opening, testing, repairing, exporting and routing PWAs, HTML projects, ZIP bundles, books (EPUB / PDF / DOCX / TXT / Markdown), images, audio and video. The core works without an internet connection. Optional cloud AI providers are off by default.

Install on iPad

  1. Open the Load page in Safari on iPad.
  2. Tap the Share button (the box with the up arrow).
  3. Scroll down and tap Add to Home Screen.
  4. Tap Add. The Load icon appears on your Home Screen and now opens like an app.

Force a fresh build

If the version number on screen is older than the latest, the cached service worker is holding the old build. To force a refresh:

  1. Open Load.
  2. Tap the circular arrow icon in the top bar (titled Force refresh — clears cache and reloads).
  3. Wait for the page to reload. The version badge in the corner should bump.

If that doesn't work, fully close Load (swipe up in the App Switcher), then reopen. Last resort: delete the icon and re-add to Home Screen.

Import a file

  1. From the home screen tap Import, or open the workspace and tap a file-type tile (HTML, PWA / ZIP, PDFs, Books, Manuscript, Media, Edit Video, Workspace).
  2. Pick from On My iPad, iCloud Drive, Dropbox, Google Drive or OneDrive.
  3. Wait for the spinner. PDFs and EPUBs need a few seconds to parse.
  4. The file lands in the Library (your local file shelf).
Kindle books: .azw / .mobi / .kfx are DRM-locked. Convert to EPUB on a computer first using Calibre.

Library & search

  1. Tap the Library tab in the top bar.
  2. Tap the magnifying-glass icon to open the search bar.
  3. Type any keyword. Search runs across the file name, your notes, the file kind (pdf / epub / html / zip / media), the folder name, and a preview of the inlined HTML.
  4. Tap the x Clear button on the search bar to reset.
  5. Use the chips below the search bar to filter by file type or folder.
If the search bar doesn't open: hard refresh (above), then try again. The button toggles a hidden bar; if your version is stale the bar element may be missing.

Preview & sandbox

  1. From the Library, tap any tile to open the file in the in-app viewer.
  2. HTML / PWA files render inside a sandboxed iframe.
  3. For an even stricter sandbox (no same-origin), open the Safety & Rights tool below and use the Reload as strict button.

Repair & diagnose

  1. From the workspace, tap Asset Doctor.
  2. It scans the open project for missing images, broken links, missing manifest, missing service worker.
  3. Tap Fix per finding to apply the suggested repair.
  4. Use Manifest to auto-build or repair manifest.json.
  5. Use Service Worker to generate an offline cache.

One-Click PWA Builder

Type a project, tap one button, get an installable PWA ZIP with manifest, service worker, icons, safety scan, receipt — and now optional photos and video clips bundled in.

  1. Tap One-Click PWA Builder in workspace Section 6.
  2. Fill in Project name, Short name, Theme color, Project type (app / book / reader / checklist).
  3. (Optional) Pick a 512x512 icon. If skipped, Load generates one from the project's first letter.
  4. Type your content in the textarea. For a book, start chapters with ## Chapter title.
  5. Photos: tap the photos box, multi-select PNG/JPEG/WEBP/GIF/SVG. They land in assets/images/.
  6. Video clips: tap the clips box, multi-select MP4/WEBM/MOV. They land in assets/clips/.
  7. To embed a photo or clip inline in the body, write [image: name.jpg] or [clip: name.mp4] on its own line. Anything not inlined goes to a gallery / clips section at the bottom.
  8. Tap Build Standalone PWA ZIP. Watch the 15-step pipeline.
  9. If the safety scan flags a BLOCKER (credential / executable / hidden .env), the build stops — fix the input and rebuild.
  10. Tap Download ZIP. Unzip and host the folder, or open index.html on a local server to install as a PWA.
Open the PWA Builder

Book PWA Validator

Confirm an exported book PWA opens as a real app (not raw code).

  1. Tap Book PWA Validator in workspace Section 6.
  2. Drop the book PWA .zip file.
  3. Read each section (required files, required folders, manifest, service worker, book content, cover image, index.html, asset paths). Each row is PASS / WARN / FAIL.
  4. Tap Save export receipt to log this validation in the Receipts library.
  5. Tap Download JSON report to keep the full report.
Open the Book PWA Validator

Export receipts

Every export logged through window.LoadReceipt.save() appears here.

  1. Tap Export Receipts in workspace Section 6.
  2. The library lists every saved receipt with status pills (manifest / service worker / rights / offline / safety) and the next recommended action.
  3. Tap a receipt's Download JSON to keep the proof.
  4. Tap New receipt to log a manual entry for a legacy export that didn't auto-generate one.
  5. Tap Export all (JSON) to dump everything in a single file.

AI Provider Status

  1. Tap AI Provider Status.
  2. Each row shows one of six pills: Not configured / Ready / Failed / Rate limited / Unsupported request / Returned no file.
  3. Paste an API key in the password field, tap Save key.
  4. Tap Test. The pill flips to Ready (or to Failed with the HTTP status, or to Unsupported if the browser blocks the request).
  5. If the browser CORS-blocks a real working key (common on iPad Safari for some endpoints), tap Mark Ready after you confirmed the key works through the Load AI panel itself.
Lock rule: Load never claims an image or audio was generated unless an actual file blob exists. The Returned no file pill is the visible surface for that — it appears when a provider answered without a usable file.
Open AI Provider Status

Voice Diagnostics (Piper + browser TTS)

  1. Tap Voice Diagnostics.
  2. Tap each test in order: Raw audio → Browser voice → Piper generate → Piper play. Each step reports PASS / FAIL / SKIP with the actual error text.
  3. If Piper is not installed, the Piper tests show SKIP and the explanation says Piper not installed or not connected — this is correct, not a bug.
  4. To install Piper: open Load Settings → Premium voice → Piper → Install.
  5. If Piper is installed but won't play, run Reset Piper Settings, then Clear Corrupt Piper Config, then Rebuild Voice Cache. Hard Reset is the last resort — it drops localStorage piper keys, Cache Storage piper caches, and IndexedDB databases matching /piper/i.
  6. If Piper still won't play, copy the visible error from the Piper-play test and send it — Stage 1 unblock is waiting on that exact text.
Open Voice Diagnostics

  2. Drop any .zip (PWA bundle, book PWA, LoadStudio package).
  3. The tree on the left lists every file. Tap a text file (HTML, CSS, JS, JSON, Markdown, TXT) to edit it inline.
  4. Tap Save to keep the change in memory.
  5. Tap Replace next to any image / audio / video to swap the binary asset.
  6. Tap Add file to add a new file at any path.
  7. Tap Download ZIP to repack and save the modified bundle. Original file on iPad is untouched.
Open PWA File Editor

Mask Editor

  1. Tap Mask Editor in workspace Section 6.
  2. Drop a photo. The image loads onto a layered canvas.
  3. Use Brush to paint white over the area you want the AI to change. Use Erase to undo a stroke.
  4. Adjust Size, Hardness, Opacity. Tap Undo to step back, Invert to flip painted vs. untouched, Clear mask to start over.
  5. Tap Download mask PNG for just the binary mask, or Download mask + source to grab both files. White = the region you painted; black = leave alone.
  6. Hand the source + mask to any inpaint-capable AI provider (or use Load AI's ADD_OBJECT flow once it accepts a mask file).
Open Mask Editor

Project Backup

  1. Tap Project Backup in workspace Section 6.
  2. The page lists what's on this device: library count, receipts, characters, acceptance results, AI statuses, other localStorage keys.
  3. Tap Build backup ZIP for a full backup including library binary blobs, or Build lite backup for metadata only (smaller).
  4. Save the resulting load-backup-*.zip to iCloud Drive or AirDrop to another device.

Speed Reader (RSVP)

  6. Pick background tint (Cream, Yellow, Blue) and word size for comfort.
Open Speed Reader

EPUB Builder

Find & Replace

  1. Tap Find & Replace in workspace Section 6.
  2. Drop a text file or paste your manuscript into the source pane.
  3. Tap Add rule for each find/replace pair. Toggle case-insensitive, whole-word, or regex per rule.
  4. Tap Common typos preset to inject ten cleanup rules (extra space before punctuation, em-dash, ellipsis, smart quotes).
  5. Tap Preview to see a diff (red strikethrough = removed, green = added) and a per-rule replacement count.
  6. Tap Apply to commit. Undo restores the prior text. Download .txt or Copy exports the result.
Open Find & Replace

Acceptance Criteria

  1. Tap Acceptance Criteria in workspace Section 6.
  2. Tap Run automatic checks — 8 rows that verify file paths, code presence, and banned-absolute scans run instantly.
  3. Walk through the manual rows on iPad. Each says How to verify:. Do the step, then tap Mark pass or Mark fail.
  4. The dashboard never auto-passes a manual check. NOT TESTED stays NOT TESTED until you tap.
  5. Tap Export JSON to keep the proof-of-pass record.
Open Acceptance Criteria
