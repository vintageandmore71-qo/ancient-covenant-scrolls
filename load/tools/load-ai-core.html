X-AI-CORE Phase 1 — read-only routing

Load owns the orchestration layer. For each intent, Core asks the shared Provider Registry for every provider that satisfies the required capability and ranks them local-first / free-first. Cloud providers that need a paid key are off by default and only appear when the toggle below is on. Phase 1 makes routing visible — no generator is called from this page.

Master rule: MVP uses free / open-source / local-first providers only.





  Ranking weight: built-in (10) < free-api (20) < local (30) < prompt-only (40) < user-imported (50) < cloud-optional (60) < load-hosted-future (90). Status “Ready” adds 0; “Needs user setup” adds 5; “Not configured” adds 10. Lower total = preferred.
  


  Output proof rule (downstream): when a real generator is wired up, Core only marks output ready if a real playable file/blob/URL is produced. Otherwise the route ends at a prompt-only provider so the spec scene field stays labelled Sound prompt saved / Silent video.
