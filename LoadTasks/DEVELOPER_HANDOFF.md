# Developer Handoff Rules

When using Load Tasks reports with Claude or a human developer, require this final report before accepting the build.

## Required final report

- Files changed.
- New files added.
- Features completed.
- Features not completed.
- Tests performed.
- Security risks remaining.
- How to open the PWA.
- How to install the PWA.
- How to confirm offline mode.
- Whether the build opens as an app or raw code.

## Non-negotiable acceptance rule

Do not mark a build complete unless the app verifies the requested files, PWA structure, promised features, export readiness, and unresolved risks.

If a claim is not verified by file evidence, status must remain Not proven.
