User Manual

Load

Run web apps offline on iPad. Build, edit, package, share — all in your browser.

1. What is Load?

Load is an offline-first Progressive Web App suite built to run on iPad Safari. It lets you import any single-file or zipped web app, run it in an embedded preview, edit its code, manage assets, design book covers, export to PDF / EPUB / standalone HTML, and route through 17 free AI providers for image generation, editing, and chat — all without sending anything to a server unless you choose to.

Tagline: Work offline. Built for iPad. Free forever.

2. Install Load on your iPad

  1. Open Safari on your iPad and visit https://dssorit.github.io/ancient-covenant-scrolls/load/
  2. Tap the Share icon (square with arrow up) in Safari's address bar
  3. Scroll down → tap Add to Home Screen
  4. Tap Add in the upper-right
  5. The Load icon appears on your home screen — tap to launch
Storage tip: when iOS asks about persistent storage, allow it. Otherwise Safari may evict your imported apps under memory pressure.

4. Importing a web app

From any screen, tap Import:

4.1 Single HTML file

  1. Tap the HTML card
  2. Pick a .html file from Files / iCloud / AirDrop / email
  3. Load reads it and adds it to your Library — done
Best for: standalone offline apps like Load-Standalone.html, Attain-Standalone.html.

4.2 ZIP package

  1. Tap the ZIP card
  2. Pick a .zip containing an index.html at the root
  3. Load extracts it, inlines all assets, injects a runtime shim that intercepts fetches and serves from the bundle
  4. The app appears in your Library
Best for: packaged PWAs like Load-PWA.zip, Attain-PWA.zip, Study-PWA.zip.

4.3 Multi-file folder

Select multiple files from Files (an index.html + its assets) and pass them all in. Load finds the HTML and inlines the other files automatically.

4.4 Video / audio

Pick a video or audio file and Load wraps it in a player UI you can replay from your Library.

5. The Library

Every imported app gets a tile in the Library showing its name, custom icon, last-opened time, and reading-time totals.

  • Tap a tile → opens the app in Load's embedded preview iframe
  • Long-press → context menu with View, Notes, Folder, Share, Edit, Add to Home, Rename, Delete
  • Sort and filter from the Library top bar

7. Image Prompt — the AI image studio

Tap the 🎨 Image Prompt tile to open the multi-provider AI image suite.

7.1 First-time setup

  1. Tap the Set Up Free banner
  2. Pick a free chat provider (Puter is no-key default; Gemini, OpenRouter, Cerebras, HF Mistral are free with a free key)
  3. Optional: paste an HF token for image gen + image editing (HF SDXL inpainting, Real-ESRGAN, GFPGAN, CodeFormer)
  4. Tap Save & Start

7.2 Free-tier providers (17 image models)

  1. Pollinations Flux — sharp photoreal default, no key
  2. Pollinations classic — fast fallback, no key
  3. Hugging Face cascade — SDXL / SD-1.5 / FLUX (HF token)
  4. Cloudflare AI — FLUX-schnell (CF account + token)
  5. Together AI — FLUX-schnell-Free
  6. AI Horde anonymous — slow, truly free
  7. Google Imagen / Gemini 2.5 Flash Image — free with Gemini key
  8. DeepAI
  9. Pollinations Turbo
  10. AI Horde SDXL
  11. Cloudflare SDXL-Lightning
  12. HF SDXL-Turbo
  13. SiliconFlow — FLUX.1 Kontext img2img edit (opt-in, free-entry credits)
  14. Real-ESRGAN (HF) — upscale
  15. GFPGAN (HF) — face restoration
  16. CodeFormer (HF) — face restoration alt
  17. Local SD — A1111-compatible companion machine (advanced)

7.3 Generate an image

  1. Type your prompt in the text box at the bottom — "a woman on a boat at sunset"
  2. Tap ↑ Send
  3. Wait 5-30s; the chain falls back through providers if one fails
  4. Tap the result image to enlarge / save / share

7.4 Edit an existing image (Manual Mask painter)

  1. Tap the 🖼 upload icon, pick a photo
  2. Tap 🖌 to open the Mask Painter (full-screen)
  3. Paint over the area you want changed (brush size slider; eraser; clear)
  4. Tap Done — the button shows 🖌✓
  5. Type the edit, e.g. "add sunglasses" / "remove this"
  6. Tap ↑ Send — only the masked region changes
Verified working: v17e0 sunglasses test passed end-to-end on iPad Safari.

7.5 Multi-turn refinement

Every result becomes the next-turn input automatically. Toggle the 🔒 Scene pill on for strict scene preservation across turns. Toggle off to start fresh.

7.6 Test Keys diagnostic

Settings → 🔑 Key Health → Test Keys. Every saved key shows masked (hf_****abcd) with a status icon: ✅ Ready · ⚠️ Missing · ❌ Invalid · ❌ Text-only · ⏳ Rate limited · 🌐 Network blocked.

7.7 Output Receipts (audit log)

Settings → 🧾 Output Receipts. Last 20 image tasks with provider, prompt, seed, consistency mode, similarity score. The audit trail when a result looks wrong.

9. Tips for everyday use

  • It's offline-first: after first install, every imported app works without internet
  • Reading timer: Library tracks how long you've spent in each app
  • Reading position: Load auto-saves your scroll position per app
  • Voice reader: tap any text in an open app to start TTS playback
  • Long-press tiles for the full action menu
  • Backups: Import → Export → PWA/ZIP creates a portable backup

10. Troubleshooting

ProblemFix
Importing a zip just shows codeMake sure you used Import inside Load — opening a zip from iOS Files just unpacks it. Pick the ZIP card in Import.
App stuck on stale buildSettings → Force Refresh. Or remove from Home Screen and re-add.
iCloud sync not showing importsLibrary is local to the device. Use Export → Standalone HTML to share manually.
Image gen says "All providers failed"Open Test Keys to check which keys are saved. With no keys, only no-key Pollinations and AI Horde anonymous are available.

Appendix A — Build plan summary (post-v17e9)

Comprehensive multi-source build plan that drives the Image Prompt feature, distilled into a single roadmap.

Hard rules in code

  • Never send image-editing requests to text-only models (HF Mistral / OpenRouter / Cerebras blocked from images at three layers)
  • Never claim an edit succeeded unless an actual image file/blob/URL is returned
  • Edits route to img2img / inpainting only; no silent text-to-image fallback for edits
  • Capability-based routing — every provider has explicit flags
  • Every provider call has a hard timeout (45/60/120 s) with AbortController
  • Loading state always clears in finally{}
  • Cache strings only go forward, never backward
  • Surgical revert only — never blanket reset
  • No iPad-side WASM or large-model loads without a strict memory test first

Shipped to code (v17dn through v17e9)

  • 17 image providers with capability flags
  • 6 chat providers with text-only locks
